Pycharm 远程连接服务器

Pycharm 专业版才支持连接服务器，如果需要，可以看看这个：Pycharm 破解。

1. 新建 SFTP

在专业版 pycharm 中：Tools -> Deployment -> Configuration

会出来这样的界面：

点击就 “+” 新建一个 SFTP，并命名为 serive（根据个人喜好），SFTP 是为了将本地文件传到服务器上，毕竟模型是要到服务器上训练的：

点击 SSH configuration 旁边的三个点，并新建SSH configuration：

Host 就是你服务器的ip地址，注意端口号。

User name 就是登录服务器的用户名（一般来说学校的服务器是多人使用的，每个人有自己的用户名和密码，用户名为root是管理员的权限了），

Password 就是你用户名对应的密码。

这些项填写完成后，点击 Test Connection 按钮测试是否能连接，连接成功会出现相应的弹窗：

若连接失败，就要排查 ip 地址，端口号，用户名和密码的正确性，并且要保证你电脑处于联网状态。

接下来要配置 Mappings ，Local path 是你本地项目的路径，Deployment path 是你想上传到服务器的路径:

Exclude Paths 是设置不想上传到服务器的文件的路径。

2. 配置解释器

因为要用到服务器上的 python 解释器，需要配置解释器，依次进入：File->Settings->Project->Project Interpreter:

点击 add Interpreter 添加解释器，选择 On SSH 项：

在弹出窗口中选择 Existing，再点击 …，选择我们之前创建的 configration

正常情况下是可以点击 Next 按钮的，但是当创建的 configuration 第一次被使用时, Pycharm 会提示需要将其加入 “IDE setting” 中，点击 Move 即可，然后点击 Next。

接下来在服务器中输入命令 which python ，找到服务器中 python 解释器的位置：

注意！一定要在跑这个模型的环境下运行这个命令，我这里就错了，不应该在 base 环境下用，而是要额外创建一个独立的环境。

然后在 Pycharm 中添加解释器，由于我这里已经配置好了虚拟环境，所以直接选择 Existing：

  • Interpreter 是解释器的地址：

  • Sync folder 是你服务器上存代码的地址：

以上设置完毕后，点击 Create 创建。

最后测试一下连通性，Connected 表示链接成功了。

回到最开始的页面，点击 Apply，这样就成功为项目配置好了 Python 解释器。

3. 文件更新

在调试代码时难免会更改代码，要想本地代码和服务器代码保持一致，可以勾选：Tools->Deployment->Automatic Upload(always)，每次修改 ctrl+s保存代码时会自动更新到服务器上。
