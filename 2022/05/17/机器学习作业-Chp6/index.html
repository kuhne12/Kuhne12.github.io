Exp 2 垃圾邮件分类器

现在有很多的邮件服务器能提供垃圾邮件过滤器的功能，可以精准地分辨出垃圾邮件和非垃圾邮件。这一部分将使用 SVM 来构建一个自己的垃圾邮件分类器

本次练习提供了邮件样本 x，及垃圾邮件（y=1）和非垃圾邮件（y=0）对应的标签，我们将通过这些样本训练邮件分类器。特别地，我们需要将每一封邮件转化为一个特征向量 xxx ，接下来将介绍如何从邮件构建特征向量。

Exp 2.1 预处理邮件

在开始一个机器学习的任务之前，看一看数据集中的例子通常是很有必要的。上图是一个垃圾邮件的示例，它包含了一个超链接，一个邮件地址，数字以及金额。然而许多邮件也会包含相似的东西，而一些特殊的内容（比如特殊的超链接或特殊的金额）会和绝大部分的邮件不同。因此在预处理邮件过程中需要进行 “规范化”，以便所有的超链接和金额都能被同等对待。例如，我们可以用唯一的字符串 “ httpaddr” 替换电子邮件中的每个 URL，以表明存在一个 URL。这样做的效果是，垃圾邮件分类器可以根据是否存在任何 URL，而不是根据是否存在特定的 URL，来做出分类决策。这通常会提高垃圾邮件分类器的性能，因为垃圾邮件发送者通常会随机处理 URL，因此在新的垃圾邮件中再次看到任何特定 URL 的几率非常小。

我们需要实现以下邮件预处理的步骤：

  • 转为小写：将邮件中所有的字母都转为小写，以便忽略大小字母
  • 剥离 HTML：所有邮件中的 HTML 标签都需要被移除，因为许多邮件通常都带有 HTML 格式。
  • 规范化超链接：所有的超链接都被替代为字符串 “httpaddr”
  • 规范化邮件地址：所有邮件的地址都被替代为字符串“emailaddr”
  • 标准化金额：所有的美元符号都被替代为字符串“dollar”
  • 提取词干：将所有单词压缩为词干。比如 ”discount“，”discounts“，”discounted“ 和 ””discounting“ 都被替换为 ”discount“ 。有时候，词干提取器会从结尾去掉其他字符，所以“ include”、“ includes”、“ included”和“ includes”都被替换为“ includ”。
  • 剔除”非单词“：删除非单词和标点符号。所有空白(制表符、换行符、空格)都被裁剪为单个空格字符。

预处理的结果如下图。虽然预处理留下了单词片段和非单词，但使用这种形式进行特征提取要容易得多。

在邮件预处理结束后，我们会得到每一封邮件的单词列表。下一步就要去选择哪些单词我们需要用在邮件分类器中，哪些单词我们希望忽略的。这里我们只选择了最常出现的单词构建单词列表。由于在训练集中很少出现的单词只出现在几封电子邮件中，它们可能会导致模型过拟合。词汇表已经存放在 vocab.txt 文件夹中（如下图），这个词汇表选择了垃圾邮件语料库中至少出现100次的所有词汇，包含 1899 个词汇。

有了词汇表，我们现在可以将预处理过的电子邮件中的每个单词映射到一个单词索引列表中，该列表包含词汇列表中单词的索引。下图展示了单词映射的结果，在样例电子邮件中，单词“ anyone”首先被规范化为“ anyon”，然后映射到词汇表中的索引86。
