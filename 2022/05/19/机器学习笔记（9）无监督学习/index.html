一、聚类算法

1.1 聚类算法介绍

从这一节起，将开始学习无监督学习算法，这节介绍的聚类算法也是我们遇到的第一个无监督学习算法。不同于每个样本都带有一个 label 的监督学习数据集，无监督学习的数据集中不包含样本的标签。数据是这样的：

可以看到，数据集中并不包含标签 yyy 。

无监督学习算法要将这些不带标签的数据输入算法，让算法找到一些隐含在数据中的规律。从上例可以看出，该数据集的样本可以被分成两组分开的点集，而能找出这两 “簇” 的算法就是聚类算法。

1.2 聚类算法作用

  • 市场分割
  • 社交网络分析
  • 组织计算机集群
  • 天文数据分析

2.4 K-means 算法代价函数

在我们学过的每一个监督学习算法中，都会有对应的优化目标（代价函数），实际上 K-means 算法也有一个代价函数。

在运行 K-means 算法时，我们将会对两组变量 cic^ici 和 μk\mu_kμk​ 进行跟踪 :

  • cic^ici：表示样本 xix^ixi 所属的簇号
  • μk\mu_kμk​ ：表示第 kkk 个聚类中心的位置

还有一个符号 μci\mu_{c^i}μci​ ，用于表示样本 xix^ixi 所属聚类中心的位置，基于这些符号，就能写出 K-means 算法的代价函数了：

J(c1，⋅⋅，cm,μ1，⋅⋅，μK)=1m∑i=1m∣∣xi−μci∣∣2J(c^1，··，c^m,\mu_1，··，\mu_K)=\frac{1}{m}\sum^m_{i=1}||x^i-\mu_{c^i}||^2 J(c1，⋅⋅，cm,μ1​，⋅⋅，μK​)=m1​i=1∑m​∣∣xi−μci​∣∣2

之前讲到 K-means 算法的簇分配过程，就是在选择 ccc 值来最小化代价函数 JJJ ，保持最近的聚类中心不变，选出c1,c2,⋅⋅，cmc^1,c^2,··，c^mc1,c2,⋅⋅，cm ；而 K-means 算法的移动聚类中心过程，就是选择 μ\muμ 值来最小化代价函数 JJJ ，保持聚类样本不变，选出 μ1，⋅⋅，μK\mu_1，··，\mu_Kμ1​，⋅⋅，μK​ 。相当于把优化过程分为了两步，一部分优化 ccc ，一部分优化 μ\muμ，随后保持迭代。

2.6 选取聚类数量 K

对于选取聚类数量 K，其实没有很好的办法，最常用的办法仍然是观察可视化的图，或者分析聚类算法的输出。

选择聚类的数量并不容易，很大程度上是因为人们通常不清楚数据集中有多少个类，对于以下数据集，

有时候答案不止一个，因为数据没有 label，所以并不总是有一个明确的答案，所以用一个自动的算法来决定聚类的数量是很困难的。

  • 肘部法则

    在谈论到选择聚类数量 KKK 时，可能使用“肘部法则”：改变 KKK 的数量，先用 1 个类来聚类，计算代价函数值；然后再用 2 个类来聚类，继续计算代价值，以此类推，将代价值标注在图像上，随着聚类的增多，代价值会不断下降，得到类似的图像：

    观察这幅图像，可以看到曲线像一个手臂，而“肘部”对应的 KKK 可能就是一个比较好的取值。但是不是每次都能得到这么理想的图像，有时候会得到到一条很光滑的曲线：

    这幅图并不能看到一个清晰的拐点，所以很难选取出一个恰当的值。

  • 基于后续目的选择聚类数量 K

    通常人们使用 K-means 聚类是为了得到一些类别，用于接下来的目的（市场分割、组织计算机集群等等）。如果后续目的能给予一个评估标准，那么就可以根据后续目来决定聚类的数量（比如分割 T 恤型号的问题）。

3.4 PCA 与线性回归

在介绍 PCA 时，总会出现这样的一条直线：

这看起来有点像线性回归，但事实上 PCA 不是线性回归。

  • 线性回归

    给定某个输入特征量 xxx ，希望模型给出预测值 yyy 。需要拟合一条直线，来最小化点和线之间竖直距离的平方误差，即最小化图中蓝线的总和。

  • PCA

    找到一条直线，使样本投影到直线的距离总和最短，即最小化图中蓝线的平方和，与线性回归的区别是：这里计算的是垂直于直线的距离平方和。

即便这两个例子的图像看起来差不多，但线性回归和 PCA 并不是同一个算法。

3.5 PCA 算法实现

在使用 PCA 之前，首先要进行数据预处理：

  • 均值标准化：

    首先计算每个特征的均值，随后将所有的特征减去均值，这将使所有特征的均值正好为 0 。

  • 特征缩放：

    如果不同的特征在数值上相差很大（例如 x1x_1x1​ 代表房子尺寸，x2x_2x2​ 代表房价），我们需要将特征减去均值再除以标准差，来进行特征缩放。

做完一系列的数据预处理后，将开始 PCA 算法的工作：找到一个低维子空间（KKK 维向量 uuu），对数据进行投影，并最小化投影误差平方和，得到对应维度的数据 zzz。

即需要得到向量 u∈Rn∗ku \in R^{n*k}u∈Rn∗k 和投影后的样本 zzz ：

x∈Rn→z∈Rkx \in R^n \rightarrow z\in R^k \\ x∈Rn→z∈Rk

经数学推导，证明了寻找正确的向量 u\bf uu 和投影数据 z\bf zz 的方法，但是非常复杂，这里讲一下具体的求解过程：

  1. 计算 “协方差矩阵” Σ=1m∑i=1n(xi)(xi)T\Sigma = \frac{1}{m}\sum^n_{i=1}(x^i)(x^i)^TΣ=m1​∑i=1n​(xi)(xi)T ， xi∈Rn∗1x^i \in R^{n*1}xi∈Rn∗1 ，(xi)T∈R1∗n(x^i)^T \in R^{1 * n}(xi)T∈R1∗n ，Σ∈Rn∗n\Sigma \in R^{n*n}Σ∈Rn∗n ；
  2. 使用线性代数库计算协方差矩阵的特征向量 U∈Rn∗nU \in R^{n*n}U∈Rn∗n ；
  3. 若需要将 nnn 维降至 kkk 维，需要取 UUU 的前 kkk 个向量 u1,⋅⋅⋅,uku^1,···,u^ku1,⋅⋅⋅,uk （前 kkk 列），得到降维的矩阵 Ureduce∈Rn∗kU_{reduce} \in R^{n*k}Ureduce​∈Rn∗k ；
  4. 计算 z=UreduceTx(z∈Rk)z=U_{reduce}^Tx \qquad (z\in R^{k})z=UreduceT​x(z∈Rk)

这样就得到了投影后的样本 zzz。

3.7 主成分数量 k 选择

在 PCA 算法中，我们将 nnn 维特征减少为 kkk 维特征，而这个 kkk 就是 PCA 算法的一个参数，也被称为主成分数字，这一节将介绍如何选择这个参数 kkk。

为了选择出参数 kkk ，我们需要了解几个概念：

  • 平均投影误差平方：1m∑i=1m∣∣xi−xapproxi∣∣2\frac{1}{m}\sum^m_{i=1}||x^i-x_{approx}^i||^2m1​∑i=1m​∣∣xi−xapproxi​∣∣2
  • 数据总方差：1m∑i=1m∣∣xi∣∣2\frac{1}{m}\sum^m_{i=1}||x^i||^2m1​∑i=1m​∣∣xi∣∣2

选择方法是：选择能使以上两者之间的比值小于0.01的 kkk 值，即：

平均投影误差平方数据总方差=1m∑i=1m∣∣xi−xapproxi∣∣21m∑i=1m∣∣xi∣∣2≤0.01(1%)\frac{平均投影误差平方}{数据总方差}=\frac{\frac{1}{m}\sum^m_{i=1}||x^i-x_{approx}^i||^2}{\frac{1}{m}\sum^m_{i=1}||x^i||^2} \le0.01\qquad(1\%) 数据总方差平均投影误差平方​=m1​∑i=1m​∣∣xi∣∣2m1​∑i=1m​∣∣xi−xapproxi​∣∣2​≤0.01(1%)

用 PCA 的语言来表述就是，有 99% 的方差性会被保留。

（这个可以看成有损压缩，损失的数据不超过百分之一）

具体实现过程是:

  1. 首先将 kkk 设置为 1
  2. 运行 PCA 算法，计算UreduceU_{reduce}Ureduce​、 z1,z2z_1,z_2z1​,z2​ 一直到 zmz_mzm​ 、xapprox1,xapprox2x_{approx}^1,x_{approx}^2xapprox1​,xapprox2​ 一直到 xapproxmx_{approx}^mxapproxm​
  3. 检查比值是否保留了 99% 的方差
  4. 若比值小于 0.01，就令 k=2k=2k=2 ，重复以上步骤，直到比值小于 0.01

这种步骤看起来非常低效，但是在调用线性代数库的时候，可以得到一些方便计算这个比值的数据。

调用 svd 函数后，可以得到矩阵 U,S,V ，S 是一个 N∗NN*NN∗N 的方阵，它的对角线 s11−snns_{11}-s_{nn}s11​−snn​ 是矩阵中”唯一“的非零元素。对于一个给定的 kkk ，对应的比值可以这样计算：

1−∑i=1ksii∑i=1nsii1-\frac{\sum^k_{i=1}s_{ii}}{\sum_{i=1}^ns_{ii}} 1−∑i=1n​sii​∑i=1k​sii​​

即检查 1−对角线前k个元素和对角线总和1-\frac{对角线前k个元素和}{对角线总和}1−对角线总和对角线前k个元素和​ 是否小于等于 0.01，不用重复地执行函数。

由于现实中的数据往往有内在的关联性，所以无论是多高维的数据，都能找到一个比值小于 0.01。

3.8 PCA 加速学习算法

假设有一个监督学习问题，有输入 xi∈R10000x^i \in R^{10000}xi∈R10000 （一张100 * 100的图片）和对应的 label yiy^iyi ，对于这种高维的特征向量，运行学习算法将会很慢，而使用 PCA 算法可以有效的减少数据的维度，从而使算法运行地更加高效。

加速步骤：

  1. 检查已经带有 label 的训练集，抽取出 xix^ixi ，得到无 label 的训练集 x1,x2,⋅⋅⋅,xm∈R10000x^1, x^2, ···, x^m \in R^{10000}x1,x2,⋅⋅⋅,xm∈R10000；
  2. 使用 PCA 算法得到原始数据的低维表示 z1,z2,⋅⋅⋅,zm∈R1000z^1, z^2, ···, z^m \in R^{1000}z1,z2,⋅⋅⋅,zm∈R1000；
  3. 组合成新的训练集 (z1,y1),(z2,y2),⋅⋅⋅,(zm,ym)(z^1,y^1), (z^2,y^2), ···, (z^m,y^m)(z1,y1),(z2,y2),⋅⋅⋅,(zm,ym) ；
  4. 将新训练集输入一个学习算法（逻辑回归、神经网络）， 训练完模型后，使用 hθ(z)h_\theta(z)hθ​(z) 来进行预测
  5. 如果有新的数据集，需要完成 PCA 映射，得到降维的输入 zzz ，再进行预测

注意： PCA 所作的是定义一个从 xix^ixi 到 ziz^izi 的映射，包括特征缩放和均值归一化，同时也会计算矩阵 UreduceU_{reduce}Ureduce​ ，但是这些仅是通过 PCA 取得的参数，只应该在训练集上拟合这些参数，而不能在交叉验证集上或测试集上拟合。参数拟合完成后，就可以在交叉验证集和测试集中进行这种映射，随后再进行预测。

意思是：从 xxx 到 zzz 的映射有一个建立过程和一个应用过程。建立的时候只能用训练集，训练 PCA 的参数，建立完映射就可以直接对交叉印证集以及测试集套公式了。

通过以上的方式可以进行数据的降维，而使用较低维度的数据，就可以使学习算法运行得更快。

3.9 PCA 算法的误用

  • 误用1：防止过拟合

    如果有 xi∈Rnx^i\in R^nxi∈Rn ，使用 PCA 压缩特征至 zi∈Rkz^i \in R^kzi∈Rk ，特征数量可以减少很多，而使用更少的特征比使用更多的特征产生过拟合的概率更小，有人就会认为 PCA 是一种防止过拟合的方法，但是这是一种很不恰当的 PCA 使用方式。

    使用 PCA 减少数据维度防止过拟合的效果可能也会更好，但这并不是一种解决过拟合的好方法，如果担心过拟合问题，可以添加正则项来避免。

    原因:PCA 算法并不需要使用到 label yiy^iyi ，仅使用 xix^ixi 来寻找低维的近似数据，所以 PCA 会舍弃掉一些有价值的信息，如果 99% 的方差信息被保留，这样使用 PCA 是可行的，但是添加正则项可以达到更好的防止过拟合的效果，因为正则项是用于代价函数 J(θ)J(\theta)J(θ) 中的，而 J(θ)J(\theta)J(θ) 中保留了 label 信息，所以不用担心会损失有价值的信息。

  • 误用2：过早使用 PCA

    在实现 PCA 算法之前，我们应该首先考虑使用最原始的数据 xix^ixi ，只有这么做无法达到预期目标时（时间、空间要求），才考虑使用 PCA 和 ziz^izi 。因此，在使用 PCA 之前，与其考虑减少数据维度，不如考虑抛弃 PCA 这一步，使用原始数据训练学习算法。
