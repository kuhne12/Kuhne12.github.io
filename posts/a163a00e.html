一、改进算法

1.1 改进算法的选择

曾实现了用于预测房价的正则化线性回归模型，如果将一组新的房屋样本输入此模型进行测试，发现预测值与真实值有巨大差距，应该如何去改进这个算法？

可以想到的办法有：

  • 使用更多的训练样本（有时候添加再多的训练样本也无助于提高准确率）
  • 使用更少的 feature，去掉一些低关联性的 feature 来防止过拟合
  • 添加更多的 feature，添加一些具有高关联性的 feature 来防止欠拟合
  • 添加多项式特征 (x12,x22,x1x2，etc.)(x_1^2,x_2^2,x1x2，etc.)(x12​,x22​,x1x2，etc.)
  • 增大正则化参数 λ\lambdaλ
  • 减小正则化参数 λ\lambdaλ

以上所有方式都可以用来改进算法，但是大多数时间都是凭感觉去选择方法，导致花费很多时间却没让算法得到改进。

接下来将讨论怎样去评估机器学习算法的性能，也被称为机器学习诊断法，这是一种测试的方法，执行了这种测试就能快速了解算法的问题。

1.2 评估假设函数

通常情况下，我们会为假设函数选择能使代价函数最小化的参数，有人认为得到很小的训练误差一定是一件好事， 但是我们已经知道，过小的误差会导致过拟合问题，使模型在新的训练样本上表现不佳。

那么，如何判断假设函数是否过拟合呢？

最容易想到的方法是画出假设函数 h(x)h(x)h(x) 然后进行直观观察，但是对于有很多 feature 的样本， 画图就变得很困难，甚至说是不可能。因此我们需要另一种评估假设函数的方法。

1.3 模型选择、训练集、验证集和测试集

如果需要确定模型最适合的特征，多项式次数 和正则化参数 λ\lambdaλ，我们应该如何下手呢？

这类问题被称为模型选择问题，需要将数据分为三个数据组。

我们知道，就算假设函数在训练集上表现得很好，也不能代表在新样本上会有很好的泛化能力。

如果要确定最合适的多项式次数（一次、二次、三次直到十次函数），这类似于在模型里加入一个参数 ，我们用 ddd 表示多项式次数，也就是说，除了参数 θ\thetaθ ，还有一个参数 ddd 需要用数据集来确定。

我们可以这么做：首先选择第一个模型，然后在训练集上训练模型，得到一个参数向量 θ1\theta^1θ1 ，然后再选择第二个模型，继续得到参数向量 θ2\theta^2θ2 ，直到 θ10\theta^{10}θ10 。随后对所有模型，求出他们在测试集中的误差，JTest(θ1)、JTest(θ2)J_{Test}(\theta^1) 、 J_{Test}(\theta^2)JTest​(θ1)、JTest​(θ2) ，直到 JTest(θ10)J_{Test}(\theta^{10})JTest​(θ10) ，最后选择误差最小的模型。

假设最终选了一个五次函数，为了测试这个模型的泛化能力，要衡量这个五次多项式假设模型对测试集的拟合情况。

但问题是，这样做仍然不能有效地衡量假设函数的泛化能力，由于额外的参数 ddd 是在测试集上进行拟合，参数向量 θ5\theta^5θ5 可能只在测试集上表现得很好，又在测试集上进行假设函数的评估，就显得很“不公平”。

为了解决这一问题，我们不再把数据集简单地分成训练集和测试集，而是把它分为三个部分：训练集、交叉验证集(又称为 cv 、验证集)、 测试集。 这三部分的比例为 6:2:26:2:26:2:2（也可以稍作调整），使用 mcvm_{cv}mcv​、 (xcvi,ycvi)(x_{cv}^i,y_{cv}^i)(xcvi​,ycvi​) 表示验证集中的样本。

现在已经定义了训练集，交叉验证集和测试集，对应地就可以定义训练误差、交叉验证误差和测试误差，如下所示。

完成数据集的分割后，就要用交叉验证集选择模型次数，而不是用原来的测试集。

对所有多项式，在训练集上求出使代价函数最小化的参数向量 θ1、θ2...θ10\theta^1、\theta^2...\theta^{10}θ1、θ2...θ10 后，不再去测试集上评估假设函数的误差，而是去交叉验证集上评估，计算出 Jcv(θi)J_{cv}(\theta^i)Jcv​(θi)，选择交叉验证误差最小的假设函数（即确定下 ddd 的取值），使用测试集测试模型的泛化能力。

二、 偏差与方差

机器学习算法的运行表现不理想，多半是出现了两种情况：

  • 偏差 (bias) 较大，欠拟合
  • 方差较大，过拟合

在这种情况下，弄清楚算法具有偏差问题还是方差问题还是两者都有问题，是十分重要的，只有知道问题后，才能快速地找到有效方法和途径来改进算法。

2.1 诊断偏差与方差问题

还是以线性回归为例，下图展示了模型欠拟合、适度拟合以及过拟合三种情况。

通过上节学习，我们已经弄清楚了训练集、验证集、测试集三者的概念，这对我们处理偏差和方差问题有很大的帮助。

我们使用训练集计算训练误差Jtrain(θ)J_{train}(\theta)Jtrain​(θ)，使用验证集计算验证误差 Jcv(θ)J_{cv}(\theta)Jcv​(θ)：

在训练集上，随着多项式次数增大，对训练集的拟合会越来越好，误差也会随之越来越小。

在交叉验证集上，误差随着多项式次数的增加，应该会呈现出一种先降低后增加的趋势：当多项式次数过少时，欠拟合，误差大；当多项式次数过多时，过拟合，误差增大。

这样一张图可以很好地帮助我们找出高偏差和高方差的问题：

  • 交叉验证误差大、训练误差也大：高偏差、欠拟合，多项式次数过低
  • 交叉验证误差大、训练误差小：高方差、过拟合，多项式次数过高
  • 交叉验证误差小、测试误差小：适度拟合，多项式次数合适

2.2 偏差、方差与正则化的关系

我们知道对模型进行正则化可以有效防止过拟合问题，但是正则化跟算法的偏差和方差又有什么关系呢？

假如现在要拟合这样一个高阶多项式，它的最高次数已经确定为 4：

hθ(x)=θ0+θ1x+θ2x2+θ3x3+θ4x4h_\theta(x)=\theta_0+\theta_1x+\theta_2x^2+\theta_3x^3+\theta_4x^4 hθ​(x)=θ0​+θ1​x+θ2​x2+θ3​x3+θ4​x4

为了防止过拟合，对其进行正则化：

J(θ)=12m∑i=1m(hθ(x(i))−y(i))2+λ2m∑j=1mθj2J(\theta)=\frac{1}{2m}\sum^m_{i=1}(h_\theta(x^{(i)})-y^{(i)})^2+\frac{\lambda}{2m}\sum^m_{j=1}\theta_j^2 J(θ)=2m1​i=1∑m​(hθ​(x(i))−y(i))2+2mλ​j=1∑m​θj2​

添加这个正则项可以使参数值尽可能小，正则化的求和范围取为 j=1j=1j=1 至 j=mj=mj=m ，而非从 j=0j=0j=0 开始。

下面讨论 λ\lambdaλ 取值的三种不同情况：

  • λ\lambdaλ 特别大，例如 λ=10000\lambda=10000λ=10000 ，会导致参数被惩罚的很严重，使参数大多都约等于 0 ，假设函数会接近于 hθ(x)=θ0h_\theta(x)=\theta_0hθ​(x)=θ0​ ，假设函数图像如下。因此这个函数处于高偏差，对数据集严重欠拟合。

  • λ\lambdaλ 特别小，例如 λ\lambdaλ = 0，会导致参数惩罚力度不够，导致过拟合问题。因此这个函数处于高方差，产生过拟合问题。

  • λ\lambdaλ 大小适中，假设函数可以很好的拟合数据

那么如何自动地选择出一个最合适的正则化参数 λ\lambdaλ 呢？

我们定义分别在训练集、交叉验证集和测试集上的代价函数，且先不添加正则项（注意，这里我们已经确定好了假设函数的次数，所以这里的训练集、交叉验证集、测试集与上方的模型选择问题是不一样的）：

正如图像所示：

    • 训练集误差会很小
    • 交叉验证集误差会很大，泛化能力差
  • 如果 λ\lambdaλ 过大，可能会欠拟合，高偏差，

    • 交叉验证集误差也会很大

我们可以观察到，总有一点会使交叉验证集误差尽量小，我们需要的就是取出该点对应的 λ\lambdaλ 值。

所以在高方差情况下，使用更多的训练集数据，对改进算法是有帮助的 。

四、改进方向的选择

4.1 总结改进方向

再次回到最开始预测房价的例子，我们首先使用了正则化的线性回归模型拟合数据，但是发现并没有达到我们预期的效果，我们也提出了如下的改进方案：

  • 使用更多的训练样本（有时候添加再多的训练样本也无助于提高准确率）
  • 使用更少的 feature，去掉一些低关联性的 feature 来防止过拟合
  • 添加更多的 feature，添加一些具有高关联性的 feature
  • 添加多项式特征 (x12,x22,x1x2,...)(x_1^2,x_2^2,x_1x_2,...)(x12​,x22​,x1​x2​,...)
  • 增大正则化参数 λ\lambdaλ
  • 减小正则化参数 λ\lambdaλ

接下来进行逐一讨论。

要判断使用哪种方法，首先要绘制出学习曲线，确定模型具有高方差还是高偏差问题

  1. 使用更多训练样本

    • 高方差问题有效

    • 高偏差问题无效

  2. 使用更少的 feature

    • 高方差问题有效
    • 高偏差问题无效

  3. 添加更多的 feature

    • 高方差问题无效
    • 高偏差问题有效，一般是因为设计的假设函数太过简单

  4. 添加多项式特征

    • 高方差问题无效

    • 高偏差问题有效

  5. 增大正则化参数 λ\lambdaλ ，减小多项式系数，函数变简单，防止过拟合

    • 高方差问题有效
    • 高偏差问题无效

  6. 减小正则化参数 λ\lambdaλ ，增加多项式系数，函数变复杂，防止欠拟合

    • 高方差问题无效

    • 高偏差问题有效

4.2 神经网络架构的选择

在用神经网络拟合数据时，首先可能会选择一种相对比较简单的神经网络模型，使隐藏单元尽可能少，这样的神经网络参数不会有很多，容易出现欠拟合问题，而这种网络的最大优势是计算量很小。

与之相对的另一种情况是拟合较大型的神经网络结构，如增加隐藏层单元数量或增加隐藏层层数，这样的神经网络参数一般会有很多，计算量会非常大，但这不是主要的问题，问题是这种网络结构更容易出现过拟合，但可以通过正则化来修正。

对于隐藏层层数的选择，也可以将数据集分割为训练集、验证集和测试集，分别训练单隐藏层、二隐藏层、三隐藏层的神经网络，最后选取验证误差最小的网络结构。
