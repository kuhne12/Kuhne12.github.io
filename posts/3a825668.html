知识蒸馏原理及综述

一、知识蒸馏原理

知识蒸馏概述

化学中的蒸馏指的是将混有杂质的水加热至蒸发，然后经过冷凝，取得没有杂志的水，机器蒸馏做的其实是同一种事情，即把混有杂质的“知识”提取为小而纯的“知识” 。

如上图所示，知识蒸馏就是将大模型（Teacher）中的知识浓缩到小的模型（Student）里。教师模型比较庞大、臃肿，是许多个模型的集成，性能也相对较高，而学生模型是比较轻量化的模型，但是性能相对没有那么好。

在现代人工智能社会，各种算法（NLP、CV）其实是非常庞大的，而真正落地的终端（手机、手表、智能电视）算力非常有限。教师模型是通过海量算力、数据、电力训练出来的臃肿模型，要想将这种强大的模型部署在算力受限的终端上，必须将大模型蒸馏为小模型。

近期人工智能领域有一个非常前沿的方向：“预训练大模型”。它是指用海量语料训练出的参数量巨大的算法，性能非常强。

比如在 NLP 中，从 BERT 到 GPT-3，模型的规模在指数级变大，参数数量几乎每年增加十倍，但终端的算力却没有每年增加 10 倍，一般的解决方案是将模型部署在云端上，通过网络远程访问，如果想将模型部署在离线的终端设备上，只有想办法将模型变小。

所以，轻量化神经网络是一个非常有发展前景的领域，它分为四个主要的技术路线：

  • 压缩已训练好的模型
    • 知识蒸馏
    • 权值量化
    • 剪枝
      • 权重剪枝
      • 通道剪枝
    • 注意力迁移
  • 直接训练轻量化网络
    • SqueezeNet
    • MobileNet
    • MnasNet
    • Xception
    • EfficientNet
  • 加速卷积运算
    • im2col + GEMM
    • Winograd
    • 低秩分解
  • 硬件部署

知识的表示与迁移 （Hard & Soft targets）

如果将一匹马的图像输入一个分类网络，且该网络会输出该图像是各种类别的概率。

分为两种情况：

  1. Hard targets

    这种方式不太科学，因为这种 label 相当于是告诉网络，这就是一匹马，且它不是驴和车的程度是一样的，但是我们肉眼也能看出来，马和驴之间是有一定相似性的。

  2. Soft targets

    这种方式相对比较科学，代表网络判断这张图片有一定可能是驴，但是是汽车的概率较小。

将这两种情况对应到教师网络和学生网络：

  • 教师网络：

    应该使用 Hard targets 训练。

  • 学生网络：

    使用训练好的教师网络，对该图片进行预测，将预测结果作为 Soft targets，并用于训练学生网络。

蒸馏温度 T

虽然使用 Soft targets 保留了更多信息，训练的效率会更高，但是这些 targets 并不是特别 soft（平滑，如下图所示），我们希望将其他类别的概率也变大，将它们之间的相对大小充分暴露出来，给与学生网络区分非正确概率更强烈的信号，这时候就要引入蒸馏温度 T。

具体引入的方法是：在 Softmax 函数上做更改，将指数函数内的值都除以一个温度 TTT

qi=exp(zi/T)∑iexp(zj/T)q_i=\frac{exp(z_i/T)}{\sum_iexp(z_j/T)} qi​=∑i​exp(zj​/T)exp(zi​/T)​

  • 当 T=1T=1T=1 时，就是普通的 Softmax 函数

  • T 越大，图像就会 “越 soft”，高的值变低，低的值变高，但是相对大小不变，如下图所示

增加了温度 TTT 后，一些非正确类别的概率信息就暴露地更彻底，更有利于学生网络学习。

知识蒸馏过程

  1. 首先需要有一个已经训练好的教师网络，并将数据喂入，得到 T=tT=tT=t 时的 Softmax 值。

  2. 将同样的数据输入到学生网络中，该学生网络可能是还未开始训练的网络，或者是训练到一半的网络，再分别得到 T=tT=tT=t 和 T=1T=1T=1 时的 Softmax 值。

    这里的学生网络和教师网络不是同一个网络， S 和 T 的结构本身就不一样，一般来说 S 的结构会更轻便。

  3. 使用一个损失函数，对教师网络 (T=tT=tT=t) 得出的 soft labels，以及学生网络 (T=tT=tT=t) 得出的 soft predictions 进行处理，希望这两者尽可能地接近。

    这个损失称为 Distillation loss （蒸馏损失），或者 Soft loss，他是一个特殊的交叉熵损失函数：

    −1n∑i=1n∑j=1nyijlog(p(xij))-\frac{1}{n}\sum_{i=1}^n\sum_{j=1}^ny_{ij}log(p(x_{ij})) −n1​i=1∑n​j=1∑n​yij​log(p(xij​))

    每一项其实就是 teacher * log(student)

  4. 再用同一个损失函数，对学生网络 (T=1T=1T=1) 得出的 hard predictions 与给定的 Hard labels (y) 进行处理，也希望这两者尽可能接近。

    这个损失称为 Student loss （学生损失），或者 Hard loss，就是普通的交叉熵损失。

  5. 最终的损失是 Hard loss 和 Soft loss 的加权求和：Total loss = α x student loss + β x Distillation loss。

  6. 通过梯度下降和反向传播，微调学生网络的权重，使总损失 损失最小化

二、知识蒸馏实验结果

附带功能

将知识蒸馏应用到 MNIST 手写数字数据集上的效果非常好，而且它的作者发现它会存在一个附带功能：如果将训练学生网络的数据的一大类去除（比如将数字 3 的图像全部移除），但是在教师网络全部保留，最后学生网络还是能学到对应的那一大类，即使学生网络从来没见过这种数据。

语音识别数据集

作者同时在语音识别数据集中进行了实验，将单个模型集成十倍（用 10 个 baseline 模型）作为教师网络，再用这个教师网络训练一个更小的学生网络。最终得到的效果非常相似的，且学生网络的结构更加轻量。

使用 soft targets 的优点

作者发现，使用 Soft targets 来训练网络，可以有效防止过拟合。如下图所示：

  • 如果使用 3% 的数据集训练模型（第二行），在训练集上可以得到较高的准确率，但是在测试集上的准确率过低，即出现了过拟合。
  • 如果使用 soft targets 训练网络（第三行），即便使用了 3% 的数据，能得到使用 100% 数据差不多的效果。

三、知识蒸馏应用场景

  • 模型压缩
    • 从大的教师网络到小的学生网络
  • 优化训练、防止过拟合
    • 潜在正则化
  • 无限大、无监督数据集的数据挖掘
    • 将海量的无标签数据集喂入教师网络，将预测结果作为带标签的数据集喂入学生网络
  • 少样本、零样本学习
    • 即使训练学生网络的数据集中并不包含某一类别，通过知识蒸馏还是可以学习到该类，前提是教师网络见过。
  • 迁移知识和知识蒸馏的区别
    • 迁移学习：用不同的数据集训练网络（比如用医学影像图像训练猫狗分类器）。
    • 知识蒸馏：把一个大模型的知识迁移到另一个小模型上。

四、知识蒸馏背后的机理

一个解释

图中绿色的是教师网络的求解空间，因为教师网络较大，表达能力较强；蓝色是学生网络的求解空间 ，因为模型比较小，表达能力相对较差，包含在教师网络内。而图中红色区域是教师网络收敛的区域，黄色 是学生网络的收敛区域，它与教师网络之间有一定距离。橙色是加上知识蒸馏后，学生网络的收敛区域，相当于教师网络引导学生网络靠近自身的收敛区域。

除了 Soft targets 之外，还有一种用于处理教师网络预测结果的方法叫做 Label smoothing，但他的效果没有知识蒸馏好，因为他在处理 softmax 的结果上，只是为其它的类别添加了同一个常数概率，并没有保留其它的信息（比如更像谁、更不想像谁）。

五、知识蒸馏研究方向

  1. 教学相长

    学生网络进一步帮助教师网络成长，使教师网络达到更好的效果

    韩愈：弟子不必不如师，师不必贤于弟子。

  2. 助教

    除了 Teacher 和 Student ，我们还可以引入多个老师、同学作为助教

  3. 知识的表示（中间层）、数据蒸馏、对比学习

    这里我们提到的知识，主要通过 Soft targets 表现出来，只是网络最后一层的预测结果，但是网络的中间层（CNN 中的 feature map、attention 等）也可以拿出来做知识蒸馏。

    • Response-Based Knowledge 基于结果
    • Feature-Based Knowledge 基于特征
    • Relation-Based Knowledge 基于关系
