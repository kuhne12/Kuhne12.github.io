Git 及 GitHub 常用命令记录

1. Git 的安装

这一步就不记录了，可以在终端中输入命令 git --version 来查看本机是否安装 git。

2. 在 GitHub 中创建仓库

进入 GitHub 官网，在右上角的 “+” 号处，可以选择创建仓库，在创建仓库界面填入必要的信息，即可完成仓库的创建：

仓库创建成功后，会进入仓库的主页，这时候会出现下面的链接，后续需要使用这个链接，通过终端来提交代码

3. git 上传文件操作步骤

  1. 首先 cd 到需要上传到 github 的目录；

  2. git 仓库初始化

    这样做后该文件夹会多出一个 .git 文件，可以通过 ls -ah 命令进行查看；

  3. 将当前目录中的所有文件添加到本地缓存

  4. 查看本地缓存的文件

  5. 将本地缓存中的文件进行提交

  6. 连接远程 GitHub 服务器，并进行提交

    origin 就是远程仓库链接的别名，链接太长用起来麻烦。

    如果你的本地版本库是从远程仓库 git clone 而来，git 会默认把这个远程仓库的地址叫做 origin。 这时候依旧可以通过 git remote add 把远程仓库的名称改成 ‘xxx’，然后就可以使用这个名称来push了。

  7. 推送到远程 GitHub 服务器

4. Git 及 GitHub 常用命令

查看 git 的配置列表

git 配置

  • 配置 username

  • 配置 email

  • 添加GIT全局配置(HTTPS代理)

  • 删除 git 全局配置

    这里的 username 和 email 要换成 GitHub 对应的 username 和 email；

    --global 表示全局的，即当前用户都有效，该配置会出现在 ~/.gitconfig 文件中。

    不加 --global 就是局部配置，仅对当前仓库起效，它的配置信息会在当前仓库根目录/.git/config文件下。

    注意：局部变量覆盖全局变量！！！和编程语言里面的变量关系是一样的。

  • 配置用户名和邮箱的原因

    远程仓库需要记录提交记录是由谁来完成的，所以我们需要给本地 的git 设置用户名和邮箱，用于从本地仓库向远程仓库提交记录时，远程仓库记录下这些操作是由谁来完成的（名字和邮箱都不会进行验证）。

    配置的用户名和邮箱对 push 代码到远程仓库时的身份验证没有作用，仅仅会出现在远程仓库的commits里。

    如果配置的邮箱是 github 里真实存在的邮箱，则 commits 里显示的是这个邮箱对应的账号；如果配置的邮箱是一个在 github 里不存在的邮箱，则 commits 里显示的是你配置的用户名。

GitHub 公钥和私钥

本地和远程仓库通信时，需要认证、加密，如果每次 pull、push 时都输入密码会十分麻烦。使用 SSH 则仅需将本机生成的公钥配置到远程仓库中，上传代码时，本机使用私钥加密，远程仓库使用我们配置的公钥解密验证，这样就可以实现认证功能，下面是 GitHub 公钥的配置流程。

  • 在用户目录的 .ssh 文件中检查目录下是否有名为id_rsa.pub的文件

  • 如果已经存在且不需要新的公钥，就可以直接打开该文件，获取到公钥。

  • 如果有该文件但是需要新的，首先进行公钥备份，再创建新的公钥

  • 最后进入 GitHub 的 Settings 中的 SSH keys 进行公钥配置
