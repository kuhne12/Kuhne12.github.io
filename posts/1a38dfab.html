cs231n 学习笔记（5）训练神经网络（一）

一、明确学习方向

已经讨论过如何使用计算图来表达一个函数，且探讨过神经网络就是一种计算图，它包含若干个线性层，层与层之间通过非线性函数进行连接。

上一节也介绍了卷积神经网络，知道了它是一种特殊的网络，使用卷积层贯穿整个网络，卷积层用于保持输入图像的空间结构，是通过卷积核在输入图像上滑动计算出来的，每个卷积层有多个卷积核，每个卷积核会生成各自的激活映射，将这些激活映射堆叠起来就有了特定的深度，以保留一些空间结构。

我们需要做的是学习所有权重和参数，可以通过优化的方式更新网络参数， 比如小批量随机梯度下降：

  1. 在数据集中批量抽样
  2. 在网络上进行前向传播，得到损失值
  3. 反向传播计算梯度
  4. 使用梯度更新参数

从现在开始，我们将开始讨论训练网络过程的一些细节，包括：

  1. 构建网络的第一步

    激活函数的选择、数据预处理、权重初始化、正则化、梯度检测

  2. 训练动态

    监控学习程序、参数更新、超参数优化

  3. 模型评估

二、激活函数

2.1 非线性激活函数

在全连接层或卷积层， 将输入数据乘上权值，再将结果输入激活函数（或一个非线性单元）， 就能得到对应层的输出。在之前的例子中，我们使用过 sigmoid 激活函数和 ReLU 函数，现在我们将讨论其他不同的非线性函数，并在他们中做取舍。

2.2 Sigmod 激活函数

Sigmoid 激活函数的表达式是：

σ(x)=11+e−x\sigma(x)=\frac{1}{1+e^{-x}} σ(x)=1+e−x1​

函数图像如下：

它可以将每个元素压缩到 [0,1][0,1][0,1] 范围内，如果输入一个比较大的值， 函数的输出会十分接近于 1 ；如果输入一个比较小的值， 函数的输出会十分接近于 0。

Sigmooid 函数一度非常流行，因为它可以被看作是一种神经元的饱和放电率，但是它存在着三个问题：

  1. 饱和神经元将使梯度消失

    观察一个 Sigmoid 门，将数据 XXX 作为输入，就能得到对应的输出，如下图所示（绿框）：

    观察上游返回的梯度 ∂L∂x\frac{\partial L}{\partial x}∂x∂L​，再乘上 ∂σ∂x\frac{\partial \sigma}{\partial x}∂x∂σ​ ，这是一个局部 Sigmoid 函数的梯度，我们把它们链接起来，作为传回的下游梯度。

    • 当 X=−10X=-10X=−10 时，观察图像可以看到，该处的函数图像几乎是平的，所以局部梯度值是 0 （ ∂σ∂x≈0\frac{\partial \sigma}{\partial x} \approx0∂x∂σ​≈0 ），乘以上游梯度等于 0 ，即会使接下来传回的下游梯度变为 0 。在某种意义上说，经过链式法则后会让梯度流消失，因此 0 梯度便会传递到下游节点。
    • 当 X=0X=0X=0 时，将会得到一个合理的梯度值，并可以很好地进行反向传播
    • 当 X=10X=10X=10 时，同理，局部梯度也是 0 （ ∂σ∂x≈0\frac{\partial \sigma}{\partial x} \approx0∂x∂σ​≈0 ） ，传回的下游梯度也会为 0 ，梯度消失。

  2. Sigmoid 函数是一个非 0 中心的函数

    输入 xxx ，将其乘以某个权重矩阵 WWW ，加上偏置值 bbb ，把结果传入激活函数，即 f(∑iwixi+b)f(\sum_iw_ix_i+b)f(∑i​wi​xi​+b)

    输入神经元的数值始终为正值时，本地梯度 \frac{\part(\sum_iw_ix_i+b)}{\part w}=x ，始终为正值，所以权重矩阵 WWW 中的元素将会全是同符号的数（取决于上游梯度的符号），做参数更新时，总会朝着一个方向移动，而这种方式对于梯度更新来说是十分低效的，因为会导致梯度下降权重更新时出现z字型的下降（两个方向只能朝着 ++ 或者 – 更新，不能像蓝色箭头一样）。

    所以我们更倾向于使用均值为 0 的数据，这样就不会陷入梯度更新的问题中去。

  3. Sigmoid 函数中使用了指数函数，计算代价高，但这并不是很大的问题。

2.4 ReLU 激活函数

ReLU 激活函数在近几年变得非常流行，它的函数公式是 f(x)=max(0,x)f(x)=max(0,x)f(x)=max(0,x)，函数图像图下：

它拥有这么几个优点：

  • 在正半轴不会产生饱和
  • 相比于其他激活函数，计算成本较低
  • 收敛速度比 sigmoid \ tanh 快很多
  • 研究证明 ReLU 比 Sigmoid 更具备生物学上的合理性

但是它也是有缺点的：

  • 输出并不是以 0 中心的
  • 在负半轴会产生饱和（当 X=−10X=-10X=−10 时，梯度值为 0 ）

在负半轴出现的梯度消失现象被称为 dead ReLU ，如下图所示

图中的两条线是 w1x1+w2x2+b=0w_1x_1+w_2x_2+b=0w1​x1​+w2​x2​+b=0 在两种 www 取值下的情况，红色是因为所有数据都位于这个分割线的负区域，所以梯度为 0 没法做反向传播，绿色有部分数据位于正区域，所以能在反向传播中产生非 0 梯度。

这些 ReLu 可以在平面中的一一半区域激活，可以看到 dead ReLU 基本不再数据云里，在这种情况下，它不会被激活和更新，导致这种情况有几个原因，包括：

  • 初始化值不当
  • 学习率过高

在实际应用中，有人喜欢用较小的偏置项来初始化 ReLU （比如 b=0.01b=0.01b=0.01），以增加在初始化时被激活的可能性（实际上用的并不多）。

2.5 Leaky ReLU 激活函数

Leaky ReLU 是一个改进 ReLU 后的激活函数，它的函数表达式是 f(x)=max(0.01x,x)f(x)=max(0.01x,x)f(x)=max(0.01x,x) ，函数图像如下：

它和 ReLU 唯一的区别是，Leaky ReLU 在负轴中有一定斜率，不是保持平直，它解决了 ReLU 的很多问题：

  • 不会饱和
  • 不会有dead ReLU 现象

还有一种激活函数，称为参数整流器，简称 PReLU ，和 leaky ReLU 很像，在负区域有一个斜率，函数表达式为 f(x)=max(αx,x)f(x)=max(\alpha x,x)f(x)=max(αx,x) ，α\alphaα 是一个可以训练的参数。

它具有 ReLU 所有的优点，但是并非以 0 为中心，且有指数运算。相比于 Leaky ReLU ，ELU 在负区域存在饱和，但是这么做可能会为噪声添加一些鲁棒性。

2.7 Maxout 激活函数

Maxout 激活函数相比其它激活函数，具有不同的形式：f(x)=max(w1Tx+b1,w2T+b2)f(x)=max(w_1^Tx+b1,w_2^T+b_2)f(x)=max(w1T​x+b1,w2T​+b2​) ，并没有做数量积运算，它泛化了 ReLU 和 Leaky ReLU ，提取两个线性函数的最大值，这种方式不会饱和，所以神经元不会消亡，但是他有一个问题：他会把每个神经元的参数数量翻一倍。

在实际操作上，一般最好使用 ReLU 激活函数，要很小心的调整学习速率；也可以适当尝试 Leaky ReLu / Maxout / ELU ，但是他们的实用性较差；一般不使用 Sigmoid 。

三、预处理

3.1 数据预处理

开始解决一个机器学习问题时，都需要经过数据预处理。一般的处理方法有：零均值化数据，归一化处理（通过标准差）。

  • 零均值化

    零均值化是为了避免数据全为正或全为负（或者数据有较大偏差），导致权重优化过程缓慢。

  • 归一化

    归一化数据是为了让所有的特征在相同的值域内，使特征的贡献相同

预处理后的数据可视化如下：

实际上，大部分的图像我们都会对它做零均值化处理，但不会过多地归一化，因为对于图像来说，像素值已经在相对可比较的范围内分布了，所以没必要过于注重归一化。

除了以上零均值化数据和归一化处理，还有别的数据处理方式，比如：PCA（主成分分析）、Whitening（白化）。

但是在图像领域，我们只注重零均值化，一般也不会做这些更为复杂的数据预处理。

  • 例：一张尺寸为 [32,32,3][32,32,3][32,32,3] 的图像
    • 减去图像均值（AlexNet）
    • 减去每个颜色通道的均值（VGGNet）
    • 减去每个颜色通道的均值，再除以每个颜色通道的标准差（ResNet）

A1：测试阶段的数据也是需要做预处理的

Q2：数据零中心化处理能解决 Sigmoid 函数的不足之处吗？

A2：零中心化处理确实可以在第一层解决 Sigmoid 函数的非零中心问题，但是之后的几层并不能保证。

3.2 权值初始化

在一个标准的双（隐藏）层神经网络中，想要通过数据来学习网络中的权值，首先需要为权值赋上初值。

如果把 0 作为初始化权值会发生什么？

所有的神经元将会做同样的事（但不一定会消亡，取决于输入数据），因为权重都是 0 ，输出的数值都会相同，且会得到相同的梯度（在同一层），所以并不能把所有的权值都简单地赋为 0。

  • 想法 1：将权值赋为较小的随机数

    这样做适用于较小规模的网络，但是在深度网络中会有问题。

    在一个 6 层神经网络中进行前向传播，其中每一个隐藏层有 4096 个神经元，激活函数使用的是 tanh

    可以发现，所有的激活值都趋向于 0 ，这并不是我们想要的

    现在考虑一下反向传播，参数的梯度应该是什么样的？

    答案是梯度都是 0 ，权重不更新

    弹幕解释：前向传播时，由于权重 WWW 设置的太小，一直乘 WWW 导致每一层的输入 xxx 越来越小，而反向传播时，梯度 dWdWdW 又取决于 xxx 的大小，xxx 太小则 dWdWdW 太小，权重不更新。

    （这部分没怎么听懂，好像李宏毅讲的比较好。）

  • 想法2：增大权重，将权值赋为较大的随机数

    这样做会让所有的激活函数饱和

    将数据可视化后，可以看到数据大部分集中在 -1 和 1

    由于产生了饱和现象，所有的梯度都会趋于 0 （梯度爆炸）

  • 想法 3：Xavier 初始化

    Xavier 初始化是一种比较合理的权重初始化方法

    数据可视化后，可以看到激活函数值在所有层都有很 nice 的分布。

四、批量归一化（Batch Normalization，BN）

这部分内容的课实在是听不下去，翻译跟 shit 一样，这部分是参考了一篇知乎的文章：卷积神经网络-批量归一化。

批量归一化的主要作用是提高神经网络模型的泛化能力。

当训练数据集和测试数据集分布不同时，会极大降低网络的泛化能力，同时当训练数据集中每个 batch 的数据分布不同时，在网络训练的过程中，参数就要不断变化更新以适应新的数据分布，这样不仅会降低网络模型的训练速度，同时也会降低模型的精准性和泛化能力，为此，我们需要对数据进行归一化。

批量归一化的作用原理是在每一层网络输入前先对图矩阵求均值和方差，然后进行标准化操作，得到具有统一分布的数据，同时在标准化操作的基础上引入拉伸（Scale）参数 $\gamma $ 与偏移参数 β\betaβ , 以恢复网络层学习的特征分布。

因为输入如果全是高斯分布也会带来一些限制，让网络自己学习每个特征的分布的形态是最适合的，所以存在参数 γ\gammaγ 和 β\betaβ 。

假设输入为 x:N× Dx:N\times\ Dx:N× D、（可学习的）Scale 参数 γ\gammaγ 和偏移参数 β\betaβ

批量归一化的前向传播过程如下：

  1. 用小批量数据求均值

    μj=1m∑i=1mxi,j\mu_j = \frac{1}{m}\sum_{i=1}^mx_{i,j} μj​=m1​i=1∑m​xi,j​

  2. 求方差

    σj2=1m∑i=1m(xi,j−μj)2\sigma_j^2=\frac{1}{m}\sum_{i=1}^m(x_{i,j}-\mu_j)^2 σj2​=m1​i=1∑m​(xi,j​−μj​)2

  3. 归一化

    x^i,j=xi,j−μjσj2+ ϵ\widehat{x}_{i,j} = \frac{x_{i,j}-\mu_j}{\sqrt{\sigma_j^2+\ \epsilon}} xi,j​=σj2​+ ϵ​xi,j​−μj​​

  4. 拉伸和偏移变换

    yi,j=γjx^i,j+βjy_{i,j}=\gamma_j\widehat{x}_{i,j}+\beta_j yi,j​=γj​xi,j​+βj​

卷积神经网络中各层的输入数据分布都不同，通过批量归一化将每一层的输出数据转化为正态分布，使后续网络层不用再学习前一层的数据分布，降低了后续网络对前面网络的依赖性，提高了网络的反馈能力。批量归一化改善了网络的梯度流，减少网络训练中的梯度消失现象，可以使用较大学习率训练网络，使得网络可以更快收敛。

批量归一化一般插入到全连接层或卷积层之后，在非线性层之前

批量归一化有很多优点，包括：

  • 让深度网络更容易训练

  • 改善梯度流动

  • 允许更高的学习率，使网络收敛地更快

  • 提高网络的鲁棒性

  • 在训练中可以被视为正则化的一种方式

  • 在测试阶段没有额外的开销，可以和卷积神经网络结合

但是也有缺点：

  • 训练和测试阶段的操作不同: 这是一个常见的错误源头。在测试阶段，批量归一化是一个线性操作，可以和之前的全连接层和卷积层融合。（？？？？？什么玩意啊啊啊？）

五、学习程序流程

我们已经定义了网络结构，现在要来讨论如何监控训练过程，并在训练过程中调整超参数。

  1. 数据预处理（上面已经提到过了）

  2. 选择网络结构

    比如选择这样一个网络：

  3. 检查损失函数

    让网络进行前向传播，需要确定最终的损失函数值是否合理，由于我们使用的是 Softmax 分类器，所以可以大致估算出初始损失值，如果有 10 个类别，初始损失值就是 −log⁡(110)=2.3-\log(\frac{1}{10})=2.3−log(101​)=2.3 （另正则项为 0 ）。我们可以检查前向传播计算的损失值是否匹配，如果正确匹配了，再加入正则项，观察损失值是否有升高。

  4. 训练小数据集

    首先从很小的数据集开始训练，确保网络在这个小数据集上的损失值能接近 0（这里要关闭正则化），精确度要接近 1 。

  5. 正式开始训练网络

    完成了以上检查后，就可以开始真正的训练了。

    正式训练时，需要加上一个比较小的正则化项，首先来确定一个可以让损失值降低的学习率。

    学习率是神经网络的一个重要参数，一般需要第一个调整，当学习率过小时，会导致梯度更新速度缓慢，过大时可能会导致发散。

    学习率的粗略范围（在交叉验证集中）大概在 [1e−3,1e−5][1e-3,1e-5][1e−3,1e−5] 。

  6. 选择超参数

    选择超参数需要在交叉验证阶段进行，分为以下步骤：

    1. 选择一些离散的参数值，用几个 epoch （一次完成训练称为一个 epoch）去迭代地学习，找到一个大致的范围

    2. 细粒度地在上一步寻找到的区间内运行网络，进行进一步精确搜索

      使用 10n10^n10n 进行优化是比较好的选择，而不是均匀采样。

    问题：这仅是训练一个参数，但如果各个参数之间有影响呢？

  9. 观察学习曲线

    • 如果训练阶段的精确度和验证阶段的精确度差别很大，则说明出现了过拟合现象
    • 如果几乎没有差距，说明没有过拟合，但可以看看是否需要提高模型的泛化能力。
